# Support

This is a community-maintained Spec Kit extension. The list below is the
fastest way to get an answer.

## Before you ask

1. Read the [README](README.md) and the [WORKFLOW](WORKFLOW.md) doc.
2. Search [open and closed issues](https://github.com/d0whc3r/spec-kit-improve/issues?q=is%3Aissue).
3. Search [Discussions](https://github.com/d0whc3r/spec-kit-improve/discussions).

## Where to ask

| You want to                                              | Use                                                                                                          |
| -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| Report a bug in this extension                           | [Bug report issue](https://github.com/d0whc3r/spec-kit-improve/issues/new?template=bug_report.yml)           |
| Request a feature or behavior change                     | [Feature request issue](https://github.com/d0whc3r/spec-kit-improve/issues/new?template=feature_request.yml) |
| Propose a new `/speckit.improve.*` command               | [New command issue](https://github.com/d0whc3r/spec-kit-improve/issues/new?template=new_command.yml)         |
| Point out a documentation problem                        | [Docs issue](https://github.com/d0whc3r/spec-kit-improve/issues/new?template=docs.yml)                       |
| Ask a usage question or share a tip                      | [Discussions](https://github.com/d0whc3r/spec-kit-improve/discussions)                                       |
| Report a security vulnerability                          | [Private security advisory](https://github.com/d0whc3r/spec-kit-improve/security/advisories/new)             |
| Report an issue in Spec Kit core itself                  | [github/spec-kit](https://github.com/github/spec-kit/issues)                                                 |
| Report an issue in your AI agent (Claude, Copilot, etc.) | The agent's own support channels                                                                             |

## Response expectations

This project is maintained by volunteers. We aim to acknowledge new issues
within 5 business days. There is no service-level guarantee. Triaged issues
are labelled and grouped into the next release planning cycle.

The fastest way to get a fix shipped is a focused pull request with
reproduction steps and verification output. See [CONTRIBUTING.md](CONTRIBUTING.md).
